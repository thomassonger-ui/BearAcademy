# Instructional Design Standard

The rules every Bear Academy page must follow for content and tone. These are not suggestions.

---

## Every Page Must

- Focus on **execution**, not theory
- Answer the question: **"What does the agent DO after this?"**
- Include clear **operational context**
- Be **concise, structured, directive**

## Every Page Must Not

- Include fluff, filler, or motivational language
- Pad with background history the agent doesn't need
- Use vague instructions like "consider" or "think about"
- Lean on hype ("game-changing", "next-level", "unlock")

---

## Tone

- **Direct** — say the thing, don't dress it up
- **Operational** — focus on the action and the system behind it
- **System-focused** — the agent is learning a repeatable process, not inspiration
- **No hype, no fluff**

---

## Page Pattern

Most pages should follow this flow:

1. **Context** — one panel explaining where this fits in the agent's workflow
2. **The System** — one or more panels explaining the exact steps or rules
3. **The Numbers / The Script / The Table** — structured panel with the operational detail
4. **What You Do Next** — closing panel stating the specific next action

If a page doesn't reach "what you do next," it's theory — revise it.

---

## Writing Checks

Before publishing, confirm:

- [ ] Every paragraph has either an instruction, a number, or a decision rule
- [ ] No sentence exists purely to motivate or inspire
- [ ] The closing panel states a specific action the agent takes when they leave the page
- [ ] Jargon is defined the first time it's used (or linked to the glossary)
- [ ] A producing agent (6–15 deals/year) could act on this page without asking for clarification
